# เป๋าตุง Wash & Dry — ระบบขายหน้าร้าน

แอพ 3 หน้าหลักสำหรับร้านซักอบ: **ล็อกอิน → บันทึกรายการขาย → ใบเสร็จ** พร้อมแผงแอดมินแยกต่างหาก ใช้งานได้พร้อมกันหลายเครื่อง เพราะข้อมูลเก็บบนฐานข้อมูลกลาง (Upstash Redis ฟรี)

## ฟีเจอร์หลัก
- **ล็อกอิน 2 แบบ**: พนักงาน (ชื่อ+รหัสผ่าน ตั้งโดยแอดมินเท่านั้น) และแอดมิน (รหัสผ่านแยกต่างหาก)
- **หน้าบันทึกรายการ**: ใส่ชื่อลูกค้า (ขึ้นต้น "คุณ" ให้อัตโนมัติ), เลือกวิธีชำระเงิน (เงินสด/QR Code), น้ำยาซักผ้า/ปรับผ้านุ่ม, เครื่องซัก (เล็ก/ใหญ่ x น้ำเย็น/อุ่น/ร้อน — ราคาปกติกับราคาโปรอยู่คู่กันให้เลือก), เครื่องอบ (เล็ก/ใหญ่ + เพิ่มเวลา) พร้อมสีไล่เฉดตามอุณหภูมิ/ขนาด/เวลา ให้ดูง่าย
- **ใบเสร็จ**: โลโก้ร้าน, ที่อยู่, ชื่อลูกค้า, วันเวลา, ชื่อพนักงาน, รายการ+โปรโมชั่น (ระบุชัดว่าใช้หรือไม่ใช้), ยอดรวม, QR Code ให้ลูกค้าสแกนดูใบเสร็จออนไลน์ และปุ่ม **บันทึกเป็น JPEG** แทนการพิมพ์
- **สต็อกน้ำยารายวัน**: น้ำยาซักผ้าและน้ำยาปรับผ้านุ่มจำกัด 20 ซอง/วัน (ตั้งค่าได้) ตัดสต็อกอัตโนมัติเมื่อขาย พนักงานปรับเองไม่ได้ ต้องแจ้งแอดมินให้เติมผ่านแท็บ "สต็อกน้ำยา" รีเซ็ตกลับเป็นจำนวนตั้งต้นทุกวันใหม่ (เวลาไทย)
- **รายงานยอดขายวันนี้ (พนักงาน)**: พนักงานดูได้เฉพาะวันนี้เท่านั้น ป้องกันการแก้ไขย้อนหลัง มีปุ่มแก้ไขรายการและดาวน์โหลดใบเสร็จซ้ำต่อบิล + export Excel
- **แผงแอดมิน**: จัดการบัญชีพนักงาน, แก้ไขราคา/โปรโมชั่นทุกรายการ, แก้ชื่อร้าน-ที่อยู่, เปลี่ยนรหัสผ่านแอดมิน, ดูรายงานรวมทุกวัน/เดือน/ปี
- **PWA**: ติดตั้งเป็นแอพบนมือถือได้ (กด "เพิ่มลงหน้าจอโฮม")
- **แจ้งเตือน LINE** (ถ้าตั้งค่า): ส่งรายละเอียดบิลเข้า LINE ทันทีที่บันทึกการขาย

## ขั้นตอนติดตั้ง

### 1. อัปโหลดโค้ดขึ้น GitHub
สร้าง repository ใหม่ที่ https://github.com/new แล้วอัปโหลดไฟล์ทั้งหมดในโฟลเดอร์นี้ขึ้นไป

### 2. นำเข้า Vercel
ไปที่ https://vercel.com/new เลือก repository นี้ กด **Import** (ค่าเริ่มต้นปล่อยไว้ ยังไม่ต้อง Deploy)

### 3. เพิ่มฐานข้อมูล Upstash Redis
ในหน้าโปรเจกต์ → แท็บ **Storage** → **Create Database** → เลือก **Upstash** → **Redis** → ตั้งชื่อแล้วสร้าง (ตัวแปร `UPSTASH_REDIS_REST_URL` และ `UPSTASH_REDIS_REST_TOKEN` จะถูกเชื่อมให้อัตโนมัติ)

### 4. (ไม่บังคับ) ตั้งค่า LINE Bot แจ้งเตือน
1. ไปที่ LINE Developers Console (developers.line.biz/console) → สร้าง Provider → สร้าง **Messaging API Channel**
2. ในหน้า Channel เลือกแท็บ **Messaging API** → กด Issue ที่ Channel access token (long-lived) → คัดลอกค่านี้ไปใส่ตัวแปร `LINE_CHANNEL_ACCESS_TOKEN` ในหน้า Vercel → Settings → Environment Variables
3. หา User ID หรือ Group ID ที่จะรับแจ้งเตือน แล้วนำไปใส่ตัวแปร `LINE_TARGET_ID`
4. ถ้าไม่ตั้งค่า 2 ตัวแปรนี้ แอพจะยังใช้งานได้ปกติทุกอย่าง เพียงแต่จะไม่มีข้อความแจ้งเตือนเข้า LINE

### 5. Deploy
กด **Deploy** รอ 1-2 นาที จะได้ลิงก์แอพ เช่น `https://paotoong-wash-dry.vercel.app`

### 6. ตั้งรหัสผ่านแอดมินครั้งแรก
รหัสผ่านแอดมินเริ่มต้นคือ `admin1234` — เข้าสู่ระบบด้วยแท็บ "แอดมิน" แล้วรีบไปที่ **เปลี่ยนรหัสผ่าน** เพื่อตั้งรหัสผ่านใหม่ทันที

### 7. เพิ่มพนักงาน
ในแผงแอดมิน → **จัดการพนักงาน** → ใส่ชื่อ + ตั้งรหัสผ่าน → บันทึก จากนั้นพนักงานใช้ชื่อ-รหัสผ่านนี้ล็อกอินหน้าแรกได้เลย

### 8. ติดตั้งเป็นแอพบนมือถือ (PWA)
เปิดลิงก์แอพในเบราว์เซอร์มือถือ → เมนู → เพิ่มลงในหน้าจอหลัก (Add to Home Screen) จะได้ไอคอนแอพเป๋าตุงบนมือถือ เปิดใช้งานได้เหมือนแอพจริง

## หมายเหตุด้านความปลอดภัย
ระบบล็อกอินนี้ออกแบบมาสำหรับใช้งานภายในร้านเท่านั้น รหัสผ่านเก็บแบบเข้ารหัส (bcrypt) ในฐานข้อมูล แต่การยืนยันสิทธิ์แอดมินฝั่งเบราว์เซอร์ใช้การเก็บรหัสผ่านชั่วคราวใน session ของอุปกรณ์นั้น ๆ (หายไปเมื่อปิดแท็บ) เพียงพอสำหรับควบคุมการเข้าถึงในร้าน แต่ไม่ควรใช้รหัสผ่านซ้ำกับบัญชีสำคัญอื่น
